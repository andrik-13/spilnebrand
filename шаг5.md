# Шаг 5 — Telegram бот

**Кто делает:** я  
**Срок:** 2 дня  
**Цель:** бот принимает заказы с сайта, уведомляет тебя, подтверждает клиенту

---

## 5.1 Как это работает (схема)

```
Покупатель на сайте
       ↓
Выбирает товар + размер
       ↓
Нажимает "ЗАМОВИТИ У TELEGRAM"
       ↓
Открывается Telegram с ботом @spilne_bot
       ↓
Бот: "Привіт! Ти хочеш замовити: Футболка базова, розмір S — 890₴. Підтверджуєш?"
       ↓
Покупатель: [Так, підтверджую]
       ↓
Бот задаёт вопросы по очереди:
  1. Ім'я та прізвище?
  2. Номер телефону?
  3. Місто?
  4. Нова Пошта чи Укрпошта?
  5. Номер відділення?
       ↓
Бот: "Дякуємо! Ми зв'яжемося з вами протягом 24 годин"
       ↓
Тебе (владельцу) приходит уведомление в Telegram
с полными данными заказа
```

---

## 5.2 Создание бота

1. Открой Telegram → найди `@BotFather`
2. Напиши `/newbot`
3. Имя бота: `SPIL'NE Shop`
4. Username: `spilne_bot` (или что-то похожее, если занято)
5. BotFather даст токен: `123456:ABC-DEF...`
6. Сохрани токен в `.env.local` как `TELEGRAM_BOT_TOKEN`

**Также нужно:**
- Создай Telegram канал/группу для получения заказов
- Добавь бота в эту группу как администратора
- Скопируй ID группы в `.env.local` как `TELEGRAM_ADMIN_CHAT_ID`

---

## 5.3 Ссылка с сайта на бота

Когда покупатель нажимает кнопку "ЗАМОВИТИ" — открывается Telegram с предзаполненными данными:

```
https://t.me/spilne_bot?start=tshirt-white_S
                                  ↑          ↑
                               slug       размер
```

Бот при получении `/start tshirt-white_S`:
1. Парсит параметр: товар = `tshirt-white`, размер = `S`
2. Делает запрос к нашему API: `GET /api/products/tshirt-white`
3. Получает название и цену
4. Показывает покупателю подтверждение

## 5.5 Обработка ошибок

| Ситуация | Реакция бота |
|---|---|
| Неверный формат телефона | "Будь ласка, введи номер у форматі +380XXXXXXXXX" |
| Покупатель нажал "Скасувати" | "Добре! Якщо захочеш замовити — повертайся 😊" |
| Товар не найден в базе | "На жаль, цей товар вже недоступний. Переглянь наш каталог: spilne.ua/catalog" |
| Бот завис / долгий ответ | Typing indicator (бот "печатает") максимум 3 секунды |

## 5.6 Техническая реализация

Бот работает через **webhook** — Telegram отправляет сообщения на наш сервер.

**Файл:** `app/api/telegram/webhook/route.ts`

```typescript
// Telegram шлёт POST запрос на:
// https://spilne.ua/api/telegram/webhook

// Мы обрабатываем и отвечаем через Telegraf
```

**Состояние разговора** хранится в памяти (Map) или в Supabase:
```
userId → { step: 'waiting_name', orderData: { product, size } }
```
